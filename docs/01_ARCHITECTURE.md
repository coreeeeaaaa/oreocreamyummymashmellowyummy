# Architecture

Oreo Cream Yummy Marshmallow Yummy is organized as a cross-checking harness. It does not ask one document to judge itself. It separates intake, scope, claims, evidence, maturity, counterexamples, redaction, and final verdict.

## Always-On Spine

1. Intake Normalizer
2. Scope Freeze Gate
3. Claim Atomization Gate
4. Layer-Time-Modality Separation Gate
5. Evidence Binding Gate
6. SRVL Maturity Gate
7. Measurement and Criteria Gate
8. Counterexample and Boundary Gate
9. Forbidden Promotion Gate
10. Repair and Re-entry Gate
11. Public Redaction Gate
12. Final Verdict Gate

## Conditional Modules

Use these only when the target requires them:

- Context Priority Module
- Target Type Module
- Axis Coordinate Module
- Logic Flow Graph Module
- Modal Impossibility Module
- Purpose Binding Module
- Design Generation Module
- Adversarial Audit Module
- Loop Controller Module

Conditional modules may add evidence, objections, or repair paths. They do not replace the always-on spine.

## Cross-Check Rule

Each gate must be checked against at least one adjacent gate:

- Scope checks Claim Atomization.
- Claim Atomization checks Evidence Binding.
- Evidence Binding checks SRVL level.
- SRVL checks Forbidden Promotion.
- Counterexample checks Measurement.
- Redaction checks Final Verdict.

No single gate owns the final verdict alone.

## Non-Goal

The harness does not guarantee truth. It prevents unsupported promotion, missing evidence, boundary collapse, and unsafe publication.
